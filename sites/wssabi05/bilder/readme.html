2. Vorraussetzungen
Um toms-galerie nutzen zu können muss ihr Webserver PHP (mit GD Bibliothek 1.6) unterstützten und genügend Speicherplatz bieten um all ihre Bilder sowie die Cachedaten und das Script selbst zu speichern. Letzteres ist allerdings nur wenige Kilobyte groß. Des weiteren kann die Galerie ohne Cache betrieben werden, was Speicherplatz spart, aber die Geschwindigkeit drastisch senkt, sowie die Serverlast extrem erhöht. (Darum nicht empfohlen!) Die Bilder müssen im Jpegformat vorliegen (bild.jpg). Bildbeschreibungen müssen im Textformat gespeichert sein und passend zum Bild (oder der Galerie) den entsprechenden Namen tragen (bild.jpg.txt).

3.Installation
Kopieren sie die Dateien index.php, thumbnail.php und form.css sowie das Verzeichnis cache und das Verzeichnis bilder (Achtung, Kleinschreibung bei Verzeichnissen bilder und cache beibehalten!) mit einem FTP-Programm auf ihren Webserver. (ggf. in ein eigenes Unterverzeichnis) Setzten sie nun die Rechte des Verzeichnisses cache auf chmod 777 (mit den meisten guten FTP-Programmen möglich z.B. LechFTP).
Nun ist das Script in der Grundkonfiguration betriebsbereit.

4. Nutzung der Galerie
Um die Galerie sinnvoll zu nutzten müssen sie nun nur noch ihre Bilder und (optional) Beschreibungen auf den Server kopieren. Das Script bietet ihnen 3 Möglichkeiten:

a) Sie zeigen alle Bilder in einer Galerie:
Dazu kopieren sie einfach ihre Bilder (bild.jpg) und die Beschreibungstextdateien (bild.jpg.txt) in das Verzeichnis bilder auf dem Server. Das Script ordnet diese nun automatisch nach Dateinamen in einer Galerie an, erzeugt die entsprechenden Thumbnails (kleine Vorschaubilder) und speichert diese beim ersten Aufruf der Galerie im Verzeichnis cache ab. Des weiteren wird zu jedem Bild die Beschreibung angezeigt, die in der entsprechend benannten Textdatei gespeichert ist.

b) Sie ordnen ihre Bilder in mehreren Galerien an:
Legen sie dazu im Verzeichnis bilder auf dem Server Unterverzeichnisse mit den Namen der Galerien an, die sie zeigen wollen (z.B. Autos Katzen Urlaub). Sie können nun ihre Galeriebeschreibungen in das Verzeichnis bilder kopieren (z.B. Autos.txt Katzen.txt). Achten Sie auf Groß- und Kleinschreibung! Kopieren sie nun ihre Bilder (bild.jpg) und die Beschreibungstextdateien (bild.jpg.txt) in das jeweilige Unterverzeichnis. Das Script ordnet nun automatisch die Galerien und die Bilder darin nach Dateinamen, erzeugt die entsprechenden Thumbnails (kleine Vorschaubilder) und speichert diese beim ersten Aufruf im Verzeichnis cache. Des weiteren wird zu jedem Bild die Beschreibung angezeigt, die in der entsprechend benannten Textdatei gespeichert ist.

c) Sie haben Bilder und Galerien im Verzeichnis Bilder:
Es werden in der Galerieübersicht zusätzlich die Bilder angezeigt, die sich im Verzeichnis bilder befinden. Alles weitere wie in Punkt a und b.

Wenn sie einzelne Bilder, oder ganze Galerien (Verzeichnis) vom Server löschen werden diese auch nicht mehr in der Galeie angezeigt.
Tip:
Wenn sie Bilder löschen oder tauschen sollten sie unbedingt auch den Cache leeren! (Inhalt des Verzeichnisses cache löschen.)

5. Anpassen an die eigenen Bedürfnisse:
Ein großer Vorteil dieses Scripts ist seine Variabilität. So können sie das Aussehen und die Funktion des Scripts ihrer Homepage anpassen.

a) Farbe und Form:
Schriftfarben, Hintergrundfarben, Rahmenfarben, Schriftarten und Schriftgrößen, sowie evtl. Hintergrundbilder lassen sich in der CSS-Datei anpassen. (Öffnen sie dazu die Datei form.css in einem Texteditor und passen diese entsprechend an. Grundlegende Kenntnisse in css vorausgesetzt. Informationen dazu finden sie unter http://www.teamone.de/selfhtml ) Rahmen und dessen Stärke, Anzahl der Spalten, Größe der Thumbnails, sowie die Bildgröße im Layout, Anzeige von Dateinamen und Bildgröße und weitere variable Anzeigemöglichkeiten lassen sich im Konfigurationsteil des Scripts anpassen. (Öffnen sie dazu die Datei index.php in einem Texteditor und folgen sie den Anweisungen im Konfigurationsteil am Begin der Datei.)

b) Layout anpassen / eigenen HTML-Code einfügen:
Viele Layoutanpassungen lassen sich bereits in der Konfiguration, bzw. per css realisieren. (Anzeigeoptionen von Titel, Bildnamen und anderen frei editierbaren Elementen (wie Links), sowie in Punkt a beschriebene Details.) Des weiteren besteht die Möglichkeit eigenen HTML-Code in das Script einzufügen. Um dies zu erleichtern habe ich die Stellen, wo dies möglich ist eindeutig im Code ausgezeichnet und etwas Platz gelassen. Nachfolgend möchte ich nun noch einmal die Stellen nennen:
HTML-Kopf: Dort können eigene Megatags eingefügt werden (für Suchmaschinen)
Oberer Teil der HTML-Seite (oberhalb der Galerie)
Unterer Teil der HTML-Seite (unterhalb der Galerie)
Es ist so auch möglich die Galerie in eine eigene Tabelle einzufügen. Wobei die Zelle im oberen Teil beginnt und im unteren Teil geschlossen wird. Doch so viel sollte eigentlich jeder wissen, der vorhat in dem Script rumzuschreiben ;o)
Es besteht außerdem die Möglichkeit eigenen HTML-Code und die Metatags zur einfacherere Handhabung in externen Dateien zu verwalten. Diese Dateien sollten nur den HTML-Body enthalten (also das was zwischen und steht!)

c) Technische Anpassung an die eigene Homepage:
Öffnen sie dazu die Datei index.php in einem Texteditor und passen die Einstellungen im Konfigurationsteil an. Sie können dort Die Startseite ihrer Homepage eintragen, den Namen der Datei definieren, Bilder und Cacheverzeichnis und Einstellungen für das Einfügen der Galerie per include-Befehl.
Besonderheiten bei Content-Systemen und dem Einfügen mit include:
Der Dateiname ist der des Scripts welches die Datei per include einfügt (Also z.B. die Startseite ihrer Homepage. Weiterhin als Hauptscript bezeichnet.)
Die Dateien thumbnail.php und thumbnail2.php müssen im selben Verzeichnis wie das Hauptscript liegen! (oder sie geben in der Variable $setup[thumbnail] den Pfad relativ zum Hauptscript mit an.)
Die Datei form.css muss vom Hauptscript geladen werden, bzw. deren Inhalt in die Haupt-css-Datei eingefügt werden.
Für die Navigation im Content-System kann die Variable $setup[linkanhang] angepasst werden. Der Inhalt kann nun aus beliebig vielen auch Temprären Variablen oder SID bestehen.
